### Root Cause Analysis

The tool call timeouts occur because:
1. MCPProxy's `/chat/call-tool` requires connecting to the underlying MCP server
2. MCP servers (stdio-based) have significant startup/connection latency
3. The default `call_tool_timeout` is 1m30s, but connections don't establish in time
4. Server status checks also require connecting to each server

**Recommendation**: Implement connection pooling or keep-alive for frequently-used MCP servers.
